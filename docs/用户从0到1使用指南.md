# 用户从 0 到 1 使用指南

本文面向首次部署与使用，按“安装 → 启动 → 设置 → 任务 → 输出”的顺序说明。

## 1. 环境准备
- Python 3.10+（建议 3.11）
- FFmpeg（必需，用于音视频处理与关键帧）

安装 FFmpeg：
- macOS: `brew install ffmpeg`
- Windows: 下载安装 FFmpeg 并将 `ffmpeg` 加入 PATH

## 2. 安装依赖
```bash
# macOS
pip install -r requirements-mac.txt

# Windows
pip install -r requirements-win.txt

# (可选) 实时会议采集依赖
pip install sounddevice
```

## 3. 启动服务
```bash
python app/main.py
```
打开：`http://127.0.0.1:19080`

## 4. 首次设置（从设置页开始）
进入“设置”页面，按以下顺序配置：

### 4.1 语音模型（本地/云端/远程）
位置：设置 → 语音模型

- **本地模型**：下载 Whisper / FasterWhisper / SenseVoice / Dolphin
- **云端 / 远程**：
  - DashScope Qwen3-ASR：填写 API Key 与地域 URL
  - FileTrans：必须配置 OSS（见下）
  - Remote API：填写 Base URL / Endpoint / 鉴权方式

### 4.2 Qwen3-ASR 的 OSS 配置（仅 FileTrans）
位置：设置 → 语音模型 → 云端 / 远程

- OSS AccessKey / Secret / Endpoint / Bucket
- OSS 访问域名（公网可访问）
- 安装依赖：`pip install oss2`

### 4.3 WhisperX（说话人分离）
位置：设置 → 语音模型 → WhisperX

- 配置 Hugging Face Token（用于说话人分离）
- 选择模型尺寸与运行设备

> 注意：WhisperX 仅用于说话人分离配置，不影响普通转写。

## 5. 任务创建与使用

### 5.1 视频/音频转写
进入 “音视频转文本” 页面：
- 上传文件或填写 URL
- 选择 ASR 引擎与语言
- 启动任务并等待完成

### 5.2 会议录音上传
进入 “实时会议记录” → 上传录音：
- 选择音频文件
- 选择模型并提交

### 5.3 会议实时采集
进入 “实时会议记录” → 实时采集：
- 选择输入设备
- 开始录制并实时转写

## 6. 说话人日志（手动生成）
Qwen3-ASR 不提供说话人日志。

如果需要说话人日志：
- 进入会议详情页
- 点击“生成说话人日志”
- 使用本地模型 `models/speaker-diarization-community-1`

## 7. 输出位置
所有任务输出默认在：
`data/outputs/<task_id>/`

常见输出包括：
- 转写文本（txt / json）
- 摘要
- 思维导图
- 内容卡片
- 学习卡片（md / csv）
- 关键帧

## 8. 常见问题
- **FileTrans 报错**：检查 OSS 配置是否完整、Bucket 是否公网可访问
- **无说话人日志**：需要在会议详情页手动生成
- **FFmpeg 报错**：检查系统是否安装并在 PATH 中
